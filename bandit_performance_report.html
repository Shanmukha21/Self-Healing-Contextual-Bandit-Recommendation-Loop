The Challenge Identified: Context-Blind Recommendations

Initially, the model was learning the global popularity of items rather than making context-aware decisions. For instance, because Gaming Live Stream received high clicks globally, the model recommended it over 75% of the time for all device and time contexts. This happened because Vowpal Wabbit was keeping user features (User namespace) and action features (Action namespace) separate, missing context-specific cost evaluation.

Before: Global Popularity Bias

Morning / Mobile Context (Target: Fashion_Tips) 2.5% vs 92.5%
Evening / Desktop Context (Target: Financial_Digest) 2.5% vs 92.5%

The model generalizes recommendations globally. It is blind to the user context and defaults to the overall most popular item.

After Fix: Context-Aware Convergence (-q UA)

Morning / Mobile Context (Target: Fashion_Tips) 92.5% Correct
Evening / Desktop Context (Target: Financial_Digest) 92.5% Correct

The model successfully creates interaction features between namespaces. Recommendations converge to target context preferences!

The Fix: Workspace Namespace Interacting

We solved the context-blindness by modifying the Vowpal Wabbit initialization parameters, adding the -q UA flag. This tells the C++ learning engine to dynamically generate interaction features combining User and Action namespaces:

// main.py - Initializing Vowpal Wabbit Workspace
- vw = pyvw.Workspace("--cb_explore_adf --epsilon 0.1 --quiet")
+ vw = pyvw.Workspace("--cb_explore_adf --epsilon 0.1 -q UA --quiet")

How to Verify This Locally

  1. Make sure your FastAPI server and Vite frontend client are running.
  2. Open the dashboard at http://127.0.0.1:5173.
  3. Click the Reset Loop button in the top header. This clears the old context-blind MongoDB history and model weights.
  4. Toggle on Start Auto-Simulation.
  5. Let it run for 100+ cycles. As it logs, switch the context dropdowns to observe the live PMF. You will see the probabilities converge exactly on the user preferences (e.g., 92.5% for Fashion Video on Morning/Mobile context).
